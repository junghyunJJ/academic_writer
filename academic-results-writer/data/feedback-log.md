# Feedback Log

Tracks reviewer corrections and user feedback to drive continuous improvement.

**Format**: YAML entries with structured feedback data.

---

## Log Entries

*Entries will be appended below as reviews are completed.*

```yaml
# Template for new entries:
# - id: "feedback-YYYYMMDD-NNN"
#   date: "YYYY-MM-DD"
#   section_id: "[reference to approved section]"
#   reviewer_assessment: "ready|minor_revisions|major_revisions"
#
#   corrections:
#     - type: "statistical|structural|style|accuracy"
#       original: "[what was written]"
#       corrected: "[what it became]"
#       lesson: "[pattern to learn]"
#
#   user_feedback:
#     satisfaction: "[1-5]"
#     comments: "[any user notes]"
#
#   patterns_to_reinforce:
#     - "[successful pattern to emphasize]"
#
#   patterns_to_avoid:
#     - "[problematic pattern to deprecate]"
```
